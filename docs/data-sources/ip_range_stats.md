# ipam_ip_range_stats (Data Source)

Reads utilization stats for an IP range by name. Use this to monitor address space consumption and detect exhaustion risk.

Stats are re-read on every `plan` and `apply`. This is intentional: the data source is designed for use in `precondition` checks and monitoring outputs where fresh values are needed.

-> This data source is separate from `data.ipam_ip_range` to avoid injecting frequently-changing stats into stacks that only need a stable CIDR reference.

## Example Usage

### Output utilization

```hcl
data "ipam_ip_range_stats" "tenant" {
  name = "prod-vpc-tenant"
}

output "tenant_utilization_pct" {
  value = data.ipam_ip_range_stats.tenant.utilization_pct
}
```

### Guard allocation with a precondition

```hcl
resource "ipam_ip_range" "my_team" {
  name       = "my-team-prod"
  range_size = 22
  domain     = var.domain_id
  parent     = var.parent_cidr

  lifecycle {
    precondition {
      condition     = data.ipam_ip_range_stats.tenant.utilization_pct < 80
      error_message = "Tenant block is over 80% utilized. Expand the parent range before allocating."
    }
  }
}
```
